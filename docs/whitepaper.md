## 💡 诞生与核心亮点：代码时代的新范式

本项目不仅是一个传统意义上的“中后台管理 + C端展示”业务系统，更是一场由 **AI 与人类结对编程（Pair Programming）完全主导** 的生产力革命试验田。

### 🤖 区别于传统开发的「降维打击」
传统软件工程从搭架构、画库表、写接口联调，往往需要两名以上、经验丰富的全栈工程师长达数周的高昂成本。而本架构做到了极限压缩：
1. **0 成本试错**：依靠 AI 的多线程理解能力，我们在没有事先绘制任何 UML 图表或 API 蓝本的情况下，直接对各个端的核心骨架进行了极速构筑。AI 就像一位不用睡觉的高级架构师，对诸如 CORS 跨域、SSR 拦截等问题在写下第一行代码前就已经完成规避。
2. **自动化运维与防墙体系**：国内特有的 GFW 拦截和包管理器 502 等复杂物理服务器难题（曾是大量独立开发者的坟墓），被 AI 连打补丁并在 Github Action 里手写内网镜像加持的 SCP 全息部署流，把原本极其痛苦的“部署上线黑盒”变成了只要 Push 代码就能自动发站的全自动“空投”。
3. **极客级审美注入**：所有界面的 CSS、玻璃态 UI 动画等审美决策并非通过现成的、枯燥的二次脚手架堆叠，而是 AI 在理解了前端最前沿的 C 端交互逻辑后手撸写出。无需繁杂的设计师交接。

## 🏗️ 三体架构全解析

本项目以超高内聚、超低耦合的策略划分为三大星系仓库，依靠 Docker 在私有云进行独立存活。

### 1. 核心指挥署：Backend (NestJS + Prisma + SQLite 后端引擎)
本架构的灵魂中枢。
- **职责**：与 SQLite 文件存储紧密打接，暴露极速 RESTful 前缀的 `/api`。
- **技术亮点**：Prisma 被深度固定在 6.4 稳定版，通过 `process.env.PORT` 提供动态伸缩。不仅能够秒级启动服务，更能零阻力横向支持 Postgres 的平替。

### 2. 星际作战舱：Frontend (Vue3 + Vite + Element Plus 管理台)
管理者操作数据的独立要塞。
- **职责**：供系统所有者（站长/运营）发布文章、审核信息、分类编辑。
- **技术亮点**：摒弃传统静态路由，依靠极其现代化的 `Vue Router` 及轻量状态管柱 `Pinia` 维护整个应用状态。在配置层采用流体布局，与 `Backend` 的通信享受全方位跨域白名单（CORS）支持。

### 3. 民用传输终端：H5 (C 端消费者超精美视界)
面向消费者（用户）的移动流媒体入口。
- **职责**：无感接收后端分发的内容，向指尖滑动的用户传递信息。
- **技术亮点**：高度移动适配（禁绝不必要的视图抖动），整体页面套用 Backdrop-filter（毛玻璃）未来悬浮设计规范。内嵌断网预热库（兜底数据 Mock），即使服务端在 1 秒前刚刚被宇宙射线摧毁，用户的前端页面依然唯美不崩塌。

## 🚀 启动与运维完全指南
本项目的所有三大分部均在你的私有服务器完美起飞，下面是从零至运行的启动说明：

### 1. 服务拉起序列 (本地模式)
> **Backend -> Frontend -> H5**
* **唤醒心脏 (Backend)**：在后端目录执行 `npm run start:dev`，随后在另一个终端运行 `npx prisma studio`，你会得到一个后台和一处完美的图形数据库查看界面 (5555端口)。
* **开启甲板 (Frontend)**：在管理台文件夹内执行 `npm run dev`，前端将绑定在默认的 5173 后端侦通频段。
* **分发 H5**：在 H5 文件夹执行 `npm run dev`，你需要在移动端或调试工具中使用，其自带了最完备的手写 UI。

### 2. 远征发版 (The CI/CD Magic)
所有仓库自带神级 `.github/workflows/deploy.yml`，你只需要赋予它们 `SERVER_HOST` 等服务器物理密钥：
* GitHub 接收指令 -> 将代码打包装箱 -> SCP 无接触空投到服务器 -> 强制切换百度云内网 Docker 镜像源拉取环境 ->  端口自动避让 (前端 80, 后端 3000, 移动端 8080) -> 分别重生为独立进程。

## 🔮 未来探索 (Roadmap)

如果资源允许，这套极致的轻架构将在未来接通以下神迹级别插件：
1. **AI 探索雷达 (AIGC 生成)**：在前端的 Admin 编辑界面集成 LLM API，让你输入标题就自动帮你补齐整篇配插图的科幻报道。
2. **智能全域缓存层 (Redis)**：目前 SQLite 对抗高并发访问还有限度，如果 H5 流量迎来千万级宇宙大爆发现象，将向 3000 端口并入 Redis 作为一层缓冲海绵。
3. **全宇宙漫游 (SSR 增强)**：若要加强面向搜索引擎的外空探索信号，未来的 H5 可能会直接平移到 Nuxt3 以保障完美的爬虫抓取能力 (SEO)。
